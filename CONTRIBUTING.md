# 📘 Guía de Contribución

Gracias por contribuir a la Python Wiki.

Lee esto antes de comenzar.

---

# 🔁 Flujo Obligatorio

## 1️⃣ Fork

Haz fork del repositorio.

## 2️⃣ Clone

git clone URL_DE_TU_FORK

## 3️⃣ Crear rama

git checkout -b doc-tema-tuNombre

Ejemplo:

git checkout -b doc-variables-juan

## 4️⃣ Hacer cambios

Editar SOLO el archivo asignado en la Issue.

## 5️⃣ Commit

git add .
git commit -m "Mejora variables - agrega ejemplos"

## 6️⃣ Push

git push origin nombre-de-tu-rama

## 7️⃣ Pull Request

Crear Pull Request hacia la rama main del repositorio original.

---

# 📌 Reglas Técnicas

- Una Issue = Un archivo = Un PR
- No modificar archivos que no estén en tu Issue
- No borrar contenido de otros estudiantes
- Mantener formato del TEMPLATE.md
- Código probado y funcional

---

# 🚫 Motivos de Rechazo

- PR demasiado grande
- Modifica múltiples archivos
- No sigue estructura
- No referencia Issue
- Mala calidad de documentación

---

# 🎯 Objetivo Pedagógico

Aprender colaboración real con GitHub.
